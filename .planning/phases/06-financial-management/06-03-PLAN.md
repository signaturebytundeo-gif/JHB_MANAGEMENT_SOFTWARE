1. Add to `src/app/actions/financial-reports.ts`:

   a. `getPnLReport(period: { year: number; month?: number; quarter?: number })`:
      - Revenue: SUM Sale.totalAmount + SUM Order.totalAmount (confirmed+ status) for the period. Use date-fns to compute date range from year/month/quarter params (startOfMonth/endOfMonth for monthly, startOfQuarter for quarterly, startOfYear/endOfYear for annual).
      - Revenue by channel: groupBy channelId on both Sale and Order, merge, join channel names.
      - COGS: For batches produced in the period, compute materials (SUM BatchMaterial.quantityUsed * RawMaterial.costPerUnit) + labor (SUM Batch.laborCostTotal) + overhead (SUM Batch.overheadCostTotal). Group by batch for detail.
      - Gross Profit = Revenue - COGS. Gross Margin % = grossProfit / revenue * 100.
      - Operating Expenses: SUM approved Expense.amount grouped by category for the period (approvalStatus IN ['auto_approved', 'approved']).
      - Net Income = Gross Profit - Operating Expenses. Net Margin % = netIncome / revenue * 100.
      - Return PnLReport type shape (see research doc).

   b. `getCashFlowStatement(period: { year: number; month: number })`:
      - Operating inflows: SUM InvoicePayment.amount where paidAt in period + SUM Sale.totalAmount where paymentMethod is cash/card (immediate cash). Label each source.
      - Operating outflows: SUM approved Expense.amount for period, grouped by category.
      - Net operating = inflows - outflows.
      - Investing/Financing: static zero for Phase 6 (no capex/debt tracking).
      - Net change = netOperating.
      - Opening/closing balance: null (no balance tracking model — show "N/A" in UI).
      - Return CashFlowStatement type shape (see research doc).

   c. `getCashFlowProjection()`:
      - Compute average daily revenue from last 30 days: total (Sale + Order revenue for last 30 days) / 30.
      - Compute average daily expenses from last 30 days: total approved expenses / 30.
      - For each of 13 weekly buckets (today + 90 days, grouped by week): projected inflow = avgDailyRevenue * 7. projected outflow = avgDailyExpenses * 7. Add outstanding invoices due in that week (Invoice where status != 'PAID', dueDate in week range).
      - Return `{ weekStart: string; projectedInflow: number; projectedOutflow: number; netCashFlow: number; cumulativeNet: number }[]`.

   d. `getWeeklyCashPosition(weeksBack: number = 8)`:
      - For each of the last N weeks: total cash inflows (InvoicePayment.amount + cash Sale.totalAmount), total cash outflows (approved Expense.amount), net position.
      - Return `{ weekStart: string; weekEnd: string; inflows: number; outflows: number; netPosition: number }[]`.

2. Create `src/app/actions/budgets.ts`:
   - `getBudgetVsActual(period: string)`: period is "YYYY-MM". Query Budget records for period. Query approved Expense records grouped by category for that month. For each ExpenseCategory: budgeted amount (from Budget, default 0), actual amount (from Expense sum), variance (budgeted - actual), variancePct. Return `{ category: string; budgeted: number; actual: number; variance: number; variancePct: number }[]`.
   - `setBudget(prevState, formData)`: useActionState pattern. Extract period, category, amount from formData. Validate with Zod (period matches YYYY-MM regex, category is valid ExpenseCategory, amount is positive number). Upsert Budget record (@@unique on [period, category]). revalidatePath('/dashboard/finance/reports').
   - `getBudgets(period: string)`: Simple findMany for a given period.

All Decimal to Number() coercion. Use date-fns for all date arithmetic.

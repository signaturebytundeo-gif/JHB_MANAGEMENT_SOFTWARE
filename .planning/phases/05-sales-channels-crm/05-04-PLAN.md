   - Modal overlay pattern (fixed inset-0)
   - In edit mode: hidden id input, pre-fill all fields including converting followUpAt date to datetime-local format
   - useEffect for toast + onClose on success

   - Props: { leads: Lead[] with assignedTo and createdBy; users: { id, name }[] }

   - "Add Lead" button opens LeadEditModal in create mode
   - Filter tabs by stage (All + each stage) — client-side useState filter
   - Table/cards with columns: Name, Company, Stage (badge), Source, Assigned To, Follow-Up Date (highlight red if overdue with "Overdue" badge), Created date
   - Stage quick-change: Select dropdown per row that calls updateLeadStage via useTransition (not full form — use startTransition + direct action call)
   - Click lead name to open LeadEditModal in edit mode
   - Follow-up overdue detection: if followUpAt < now() AND stage !== CLOSED, show red "Overdue" badge next to the date
   - Responsive: desktop table, mobile cards
   - Do NOT use drag-and-drop kanban — no DnD library is installed. Filtered table is the correct pattern.

   - Create form modal: productId (Select), name (text), discountPercent OR fixedPrice (toggle between the two), startDate (date), endDate (date)

   - Status logic: if isActive AND within date range = "Active" (green); if isActive but outside date range = "Scheduled"/"Expired" (yellow/gray); if !isActive = "Inactive" (red)
   - Deactivate button calls deactivatePromotionalPricing with useTransition

   - Add getCalculatedPrice server action to pricing.ts: wraps calculateLineItemPrice with db access, returns PriceCalculation type

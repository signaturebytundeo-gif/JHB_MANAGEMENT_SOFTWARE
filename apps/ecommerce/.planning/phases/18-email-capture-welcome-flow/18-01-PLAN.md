<task type="auto">
  <name>Task 2: Create POST /api/subscribe endpoint</name>
  <files>
    src/app/api/subscribe/route.ts
  </files>
  <action>
Create `src/app/api/subscribe/route.ts` with a POST handler:

1. Parse JSON body: `{ email: string }`
2. Validate email:
   - Return 400 if missing or empty
   - Basic regex validation: `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` — reject with 400 and `{ error: 'Invalid email address' }` if fails
3. Normalize: `email.toLowerCase().trim()`
4. Server-side dedup (simple): Use a module-level `Set<string>` to track emails sent this server lifecycle. If email already in Set, return 200 with `{ success: true, message: 'Already subscribed' }` — no error, no duplicate send. This is a lightweight guard; the client also prevents re-submission via localStorage.
5. Add email to Set, then call `sendWelcomeEmail(email)` from `@/lib/emails/send-welcome`
6. If sendWelcomeEmail returns `{ success: false }`, return 500 with `{ error: 'Failed to send welcome email' }`
7. On success, return 200 with `{ success: true, message: 'Welcome email sent!' }`

Also sync to Mailchimp if configured: after sending welcome email, call `syncCustomerToMailchimp` from `@/lib/mailchimp-sync` with the email (firstName/lastName empty since we only collect email). Wrap in try/catch — Mailchimp failure should NOT block the subscribe response.

Import pattern: follow the same style as checkout/route.ts (NextRequest, NextResponse from next/server).
  </action>

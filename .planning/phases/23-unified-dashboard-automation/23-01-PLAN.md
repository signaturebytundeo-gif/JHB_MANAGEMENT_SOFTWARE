  <name>Task 1: Create getUnifiedChannelStats server action</name>
  <files>apps/command-center/src/app/actions/channel-stats.ts</files>
  <action>
Create a new 'use server' file at `apps/command-center/src/app/actions/channel-stats.ts`.

Export type `ChannelStatItem = { channel: string; revenue: number; orderCount: number }`.

Export async function `getUnifiedChannelStats(): Promise<ChannelStatItem[]>` that:

1. Calls `await verifySession()` from `@/lib/dal`.
2. Computes `start = startOfMonth(new Date())` and `end = endOfMonth(new Date())` using date-fns.
3. Runs three parallel Prisma groupBy queries via `Promise.all`:
   - `db.sale.groupBy({ by: ['channelId'], where: { saleDate: { gte: start, lte: end } }, _sum: { totalAmount: true }, _count: { id: true } })`
   - `db.order.groupBy({ by: ['channelId'], where: { createdAt: { gte: start, lte: end }, status: { in: ['CONFIRMED', 'PROCESSING', 'SHIPPED', 'DELIVERED', 'COMPLETED'] } }, _sum: { totalAmount: true }, _count: { id: true } })`
   - `db.websiteOrder.groupBy({ by: ['source'], where: { orderDate: { gte: start, lte: end }, status: { not: 'CANCELLED' } }, _sum: { orderTotal: true }, _count: { id: true } })`
4. Fetches `db.salesChannel.findMany({ select: { id: true, name: true } })` and builds a `channelMap: Map<string, string>` (id -> name).
5. Uses a `statMap: Map<string, ChannelStatItem>` to merge all three result sets:
   - Sale rows: key is `row.channelId`, channel name from channelMap.
   - Order rows: key is `row.channelId`, channel name from channelMap. Adds to existing Sale entry if same key.
   - WebsiteOrder rows: key is the display name directly. Use `const sourceNames = { WEBSITE: 'Website (DTC)', AMAZON: 'Amazon', ETSY: 'Etsy' }`.
6. Returns `Array.from(statMap.values()).sort((a, b) => b.revenue - a.revenue)`.

Use `Number(row._sum.totalAmount ?? 0)` and `Number(row._sum.orderTotal ?? 0)` for Prisma Decimal conversion. Import: `db` from `@/lib/db`, `verifySession` from `@/lib/dal`, `startOfMonth`/`endOfMonth` from `date-fns`.

Do NOT query `getDashboardKPIs` — that would double-count revenue. This function is the standalone channel breakdown only.

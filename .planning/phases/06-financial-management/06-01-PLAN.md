1. Create `src/app/actions/expenses.ts` with these server actions:
   - `logExpense(prevState, formData)`: useActionState pattern. Validate with logExpenseSchema. Handle receipt upload: `const receiptFile = formData.get('receipt') as File | null` — if file exists and size > 0, use `put()` from `@vercel/blob` to upload to `receipts/${Date.now()}-${file.name}` with access 'public'. If BLOB_READ_WRITE_TOKEN is not set (development), skip upload and set receiptUrl to null (log warning, do not error). Lookup ApprovalThreshold by amount (same pattern as operator-orders.ts confirmOrder): findMany ordered by minAmount, find matching threshold where amount >= min and <= max (Infinity if maxAmount null). Set approvalStatus based on match.approvalType: "auto" -> "auto_approved", "single_member" -> "pending_single", "dual_member" -> "pending_dual", "dual_bank" -> "pending_bank". If auto_approved, also set approvedAt to now. Create Expense record. revalidatePath('/dashboard/finance/expenses').
   - `approveExpense(prevState, formData)`: Validate with approveExpenseSchema. Verify session user is not the expense creator (cannot self-approve). For "pending_single": set approvedById, approvalStatus="approved", approvedAt. For "pending_dual": if no approvedById yet, set first approver + status stays "pending_dual" (waiting for second); if approvedById exists and is different user, set secondApprovedById + status="approved" + approvedAt. For "pending_bank": require bankAuthorizationRef from formData, set both approvers like dual + bankAuthorizationRef + status="approved". For "reject" action: set approvalStatus="rejected". revalidatePath.
   - `getExpenses(filters?)`: Return all expenses ordered by expenseDate desc. Include createdBy (select name), approvedBy (select name), secondApprovedBy (select name). Map all Decimal fields to Number before returning. Optional filters: category, approvalStatus, dateRange.

<task type="auto">
  <name>Task 1: Bundle discount logic and cart store integration</name>
  <files>src/lib/bundle-discount.ts, src/lib/cart-store.ts</files>
  <action>
1. Create `src/lib/bundle-discount.ts` — a pure utility module:
   - Export constants: `BUNDLE_DISCOUNT_PERCENT = 10` (10% discount) and `BUNDLE_MIN_ITEMS = 3`
   - Export function `calculateBundleDiscount(items: CartItem[]): { discountAmount: number; eligibleItemCount: number; discountPercent: number }`:
     - Filter items to only "individual" products: exclude items where id starts with a known bundle prefix OR matches a bundle id (import getBundleById from bundles.ts to check). Also exclude items with `isFreeSample === true`.
     - Count total eligible item quantity (sum of quantities, not unique items — e.g., 2x Jerk Sauce 5oz + 1x Pikliz = 3 eligible items)
     - If eligible item count >= BUNDLE_MIN_ITEMS, calculate discount: sum eligible items' (price * quantity), multiply by BUNDLE_DISCOUNT_PERCENT / 100, round to nearest cent using Math.round
     - Return { discountAmount (in cents, 0 if not eligible), eligibleItemCount, discountPercent: BUNDLE_DISCOUNT_PERCENT }
   - This is a pure function with no side effects — easy to test and reason about

2. Update `src/lib/cart-store.ts` — NO changes to the store interface itself. The discount is computed in the component layer, not stored in state. This keeps the cart store simple and the discount is always derived from current items. No store changes needed.

   However, add a `bundleIds` field to CartItem interface: `bundleId?: string`. This marks items that were added as part of a pre-defined bundle so the discount logic can exclude them. When a pre-defined bundle is added to cart, the bundleId will be set to the bundle's id.

   Update the CartItem interface:
   ```typescript
   export interface CartItem {
     id: string
     name: string
     price: number
     quantity: number
     image: string
     size: string
     isFreeSample?: boolean
     isBundle?: boolean  // true if this is a pre-defined bundle item
   }
   ```
  </action>
  <verify>Run `npx tsc --noEmit` to confirm no type errors. The bundle-discount module should export correctly. Test mentally: 2x jerk-sauce-5oz (qty 2) + 1x pikliz (qty 1) = 3 eligible items => 10% off their combined subtotal.</verify>
  <done>Bundle discount pure function created. CartItem type extended with isBundle flag. Discount calculation excludes bundles and free samples, applies 10% when 3+ individual item quantity reached.</done>
</task>

  <action>
Create `apps/command-center/src/app/actions/invoices.ts` with 'use server' directive.

Implement these server actions:

1. `createInvoice(prevState, formData)`:
   - Call `verifyManagerOrAbove()`
   - Validate with `createInvoiceSchema.safeParse()`
   - Fetch the order with lineItems to calculate subtotal
   - Guard: order must be CONFIRMED or later, must not already have an invoice
   - subtotal = sum of lineItem totalPrices
   - taxAmount = subtotal * (taxRate / 100)
   - totalAmount = subtotal + taxAmount
   - issuedAt = new Date()
   - dueDate = addDays(issuedAt, 30) using date-fns — set ONCE, never recompute
   - Generate invoiceNumber via `generateInvoiceNumber()`
   - Create invoice record with status SENT (auto-send on create)
   - revalidatePath('/dashboard/finance')
   - Return { success, invoiceId }

2. `getInvoices()`:
   - Call `verifySession()`
   - Query all invoices with include: order (select orderNumber), customer (select firstName, lastName)
   - Order by createdAt desc
   - For each overdue invoice, compute current lateFeeAmount using: `principal * 0.015 * (daysOverdue / 30)` where principal = totalAmount - paidAmount and daysOverdue = differenceInDays(now, dueDate)
   - Return invoices with computed late fees

3. `getInvoiceById(invoiceId: string)`:
   - Full invoice with order (include lineItems with product), customer, payments
   - Compute current late fee if overdue

4. `logPayment(prevState, formData)`:
   - Validate with `logPaymentSchema.safeParse()`
   - Call `verifyManagerOrAbove()`
   - Create InvoicePayment record
   - Update invoice: paidAmount += payment amount
   - If paidAmount >= totalAmount: set status to PAID, paidAt = now
   - Else if paidAmount > 0: set status to PARTIAL
   - revalidatePath
   - Return { success, message }

5. `flagOverdueInvoices()`:
   - Query invoices where status NOT IN (PAID, VOID, OVERDUE) AND dueDate < now - 1 day (31 days after issue)
   - For each: update status to OVERDUE, set overdueAt = now
   - Compute and store lateFeeAmount = (totalAmount - paidAmount) * 0.015 * (differenceInDays(now, dueDate) / 30)
   - Return count of newly flagged invoices
   - This can be called from the finance page load or a cron job later

6. `getARAgingReport()`:
   - Call `verifyManagerOrAbove()`
   - Query all invoices where status NOT IN (PAID, VOID)
   - Include customer (firstName, lastName), order (orderNumber)
   - For each invoice, compute daysPastDue using differenceInDays(now, dueDate)
   - Assign bucket: CURRENT (not yet due), 1_30, 31_60, 61_90, 90_PLUS
   - Compute outstanding = totalAmount - paidAmount + lateFeeAmount
   - Return both individual invoices with buckets AND summary totals per bucket
   - Summary: { current: totalOutstanding, days1_30: total, days31_60: total, days61_90: total, days90Plus: total, grandTotal: total }

7. `getUninvoicedOrders()`:
   - Call `verifySession()`
   - Query orders where status IN (CONFIRMED, PROCESSING, SHIPPED, DELIVERED, COMPLETED) AND invoice IS null (no linked invoice record)
   - Select: id, orderNumber, totalAmount, orderDate, customer (firstName, lastName)
   - Order by orderDate desc
   - Return list of uninvoiced orders for the CreateInvoiceModal picker

Use `differenceInDays` and `addDays` from date-fns for all date math (already installed).

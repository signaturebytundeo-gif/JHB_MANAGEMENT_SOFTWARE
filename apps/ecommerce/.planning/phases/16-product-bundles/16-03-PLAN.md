1. Update the CheckoutItem interface to include `isBundle?: boolean` to match the CartItem type from cart-store.ts.

2. Update the request body parsing to accept `bundleDiscount` (number, in cents) alongside `items`:
   ```typescript
   const { items, bundleDiscount = 0 }: { items: CheckoutItem[]; bundleDiscount?: number } = await request.json()
   ```

3. Pre-defined bundle handling — the existing line_items mapping already works for bundles because a bundle is just a cart item with a name, price, and size. The line item will show:
   - name: "The Starter Bundle (2oz + 5oz + 12oz)" — uses the existing `${item.name} (${item.size})` pattern
   - unit_amount: the bundle price (e.g., 2499 for $24.99)
   - This is correct — no special handling needed for pre-defined bundles in line_items mapping.

4. Build-your-own discount handling — after building the regular line_items array, if `bundleDiscount > 0`:
   - Server-side validation: import `calculateBundleDiscount` from '@/lib/bundle-discount' and re-calculate the discount from the items array to prevent client-side tampering. Use the server-calculated discount, NOT the client-sent value.
   - If server-calculated discount > 0, add a coupon/discount. Stripe Checkout supports `discounts` with a coupon, but creating coupons on the fly is complex. Instead, use a simpler approach: add a negative-value line item is NOT supported by Stripe (unit_amount must be positive).
   - **Correct approach**: Use Stripe Checkout's `discounts` feature with an inline coupon. Create a one-time coupon via the Stripe API:
     ```typescript
     const coupon = await stripe.coupons.create({
       amount_off: serverDiscount,
       currency: 'usd',
       name: `Build Your Own Bundle (${BUNDLE_DISCOUNT_PERCENT}% off)`,
       duration: 'once',
     })
     ```
     Then add to the session: `discounts: [{ coupon: coupon.id }]`
   - Only create the coupon and add discounts if serverDiscount > 0

5. Update metadata to include bundle info:
   - Add `hasBundles: items.some(i => i.isBundle) ? 'true' : 'false'`
   - Add `bundleDiscount: serverDiscount > 0 ? serverDiscount.toString() : '0'`

6. Shipping calculation — bundles should count toward the $50 free shipping threshold like any other paid item. The existing `paidItemsTotal` calculation already sums all non-free-sample items by price * quantity, which includes bundles. No change needed for shipping.

7. Command-center compatibility notes (important context for implementation):
   - The command-center webhook at `/api/webhooks/stripe/route.ts` (in the command-center app) uses `stripe.checkout.sessions.listLineItems()` to get line items and `session.amount_total` for the order total.
   - When a Stripe coupon is applied to a checkout session, `listLineItems` returns each line item with its `amount_total` reflecting the proportionally-distributed discount. For example, if 3 items at $10 each get a 10% coupon ($3 off), each item's `amount_total` will be approximately $9.00 (Stripe distributes the $3 discount across items).
   - The `session.amount_total` field reflects the final post-discount total, so the command-center's orderTotal will be accurate.
   - Pre-defined bundles appear as regular line items with their bundle name and price — no special handling needed.
   - No command-center code changes are needed because the webhook already reads `amount_total` from both line items and the session, which Stripe adjusts for coupons automatically.

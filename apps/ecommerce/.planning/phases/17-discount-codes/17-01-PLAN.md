Create `scripts/seed-promo-codes.ts` that uses the Stripe SDK (already installed) to create two promotion codes. The script must be idempotent (safe to run multiple times).

**WELCOME10 promotion code:**
- Create a Stripe Coupon: `percent_off: 10`, `duration: 'forever'`, `name: 'Welcome 10% Off'`
- Create a Stripe Promotion Code from that coupon: `code: 'WELCOME10'`, `active: true`
- The coupon applies to the entire order (no product restrictions)

**FREESHIP50 promotion code:**
- This code gives free shipping. Stripe promotion codes on hosted checkout apply to the order total, not individual line items. Since shipping is added as a line item (not via shipping_rates), FREESHIP50 should be a fixed-amount coupon that offsets the shipping cost.
- Create a Stripe Coupon: `amount_off: 1299`, `currency: 'usd'`, `duration: 'forever'`, `name: 'Free Shipping ($50+ orders)'`
- Create a Stripe Promotion Code from that coupon: `code: 'FREESHIP50'`, `active: true`, `restrictions: { minimum_amount: 5000, minimum_amount_currency: 'usd' }`
- The $12.99 amount_off matches the standard shipping cost, effectively making shipping free

**Idempotency approach:**
- Before creating each coupon, list existing promotion codes and check if one with the matching `code` already exists
- If it exists and is active, skip creation and log "already exists"
- If it exists but is inactive, log a warning
- Use `stripe.promotionCodes.list({ code: 'WELCOME10' })` to check existence

**Script setup:**
- Import Stripe from 'stripe' (not from @/lib/stripe since this runs outside Next.js)

<task type="auto">
  <name>Task 2: Enable promotion codes on Stripe checkout session</name>
  <files>src/app/api/checkout/route.ts</files>
  <action>
Update the checkout route to enable the Stripe promo code input field at hosted checkout.

**Key Stripe constraint:** `allow_promotion_codes: true` and `discounts: [...]` CANNOT be used on the same checkout session. Stripe will reject the request.

**Implementation logic:**
- When `serverDiscount > 0` (build-your-own bundle discount applies): Keep existing behavior — use `discounts: [{ coupon: coupon.id }]`. Do NOT add `allow_promotion_codes`. The customer is already getting 10% off from the build-your-own bundle, so promo codes are not available in this scenario.
- When `serverDiscount === 0` (no bundle discount): Add `allow_promotion_codes: true` to the session. This shows Stripe's built-in "Add promotion code" field at checkout, where customers can enter WELCOME10 or FREESHIP50.

**Changes to `stripe.checkout.sessions.create()`:**
```typescript
const session = await stripe.checkout.sessions.create({
  payment_method_types: ['card'],
  line_items,
  mode: 'payment',
  phone_number_collection: { enabled: true },
  success_url: `...`,
  cancel_url: `...`,
  // Promotion codes and discounts are mutually exclusive in Stripe
  ...(discounts
    ? { discounts }
    : { allow_promotion_codes: true }),
  metadata: { ... },
})
```

This replaces the current `...(discounts ? { discounts } : {})` spread with a conditional that either applies the bundle discount OR enables promotion codes — never both.

**No other file changes needed.** Stripe's hosted checkout handles:
- Displaying the promo code input field
- Validating codes (invalid/expired shows error)
- Applying the discount to the total
- Showing the discount in the order summary

Ensure TypeScript compiles cleanly and the production build passes.

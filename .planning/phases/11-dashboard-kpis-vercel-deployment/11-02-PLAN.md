### Task 3: Update .env.example with all required Vercel env vars
type="auto"

Update `apps/command-center/.env.example` to include all required production environment variables in the correct format with clear comments. The current file has `DATABASE_URL` pointing to SQLite but production uses Neon.

Required variables for Vercel:
```
DATABASE_URL=          # Neon pooled connection string (postgresql://...)
DIRECT_URL=            # Neon direct connection string (for migrations)
SESSION_SECRET=        # 32+ character random string (generate with: openssl rand -base64 32)
RESEND_API_KEY=        # Resend API key for transactional emails
NEXT_PUBLIC_APP_URL=   # Production URL (https://command-center.vercel.app or custom domain)
STRIPE_SECRET_KEY=     # Stripe secret key (sk_live_... or sk_test_...)
STRIPE_WEBHOOK_SECRET= # Stripe webhook signing secret (whsec_...)
SLACK_WEBHOOK_URL=     # Slack incoming webhook URL (optional, enables order notifications)
WEBHOOK_API_KEY=       # Internal webhook API key for /api/incoming-order
```

Remove marketplace env vars (SQUARE, AMAZON, ETSY) from example as they are not yet implemented in production.

**Done criteria:**
- `.env.example` reflects production configuration
- All required Vercel env vars documented with comments
- File committed with message `chore(11-02): update .env.example for Vercel production deployment`
